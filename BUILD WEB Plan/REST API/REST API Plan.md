## Purpose

Expose three concentric public surfaces:

1. **Public Read API** — anonymous access to the tool registry, skills, and command catalogue. Lets anyone discover and document what BUILD MEP can do without installing anything.
2. **Demo Run API** — anonymous, rate-limited, sandboxed execution of read-only snapshot tools against curated SQLite models. Powers the homepage demos and partner integrations doing read-only queries.
3. **Authenticated Command API** — JWT-gated execution of `[RestApi]`-tagged commands. The async queue → plugin polling pattern lets external systems (clash dashboards, document control platforms, BIM coordination tools) trigger long-running Revit operations without ever touching the desktop.

Underneath, a single attribute-driven generator keeps the API contract in lock-step with MEP Bridge source. **One `[RestApi]` attribute is the only edit required to publish a new command** — the script generates the JSON registry, the OpenAPI spec, the docs page, and the Edge Function validator from it.

## 5. Risk Research — Known Issues & Pitfalls

| # | Area | Finding | Mitigation | Source |
|---|------|---------|------------|--------|
| 1 | Supabase Edge Function cold start | Deno cold start adds ~150 ms to first request after idle. | Set min instances on production functions (paid tier); cache hot path; CDN-cache GET endpoints. | Supabase docs |
| 2 | Async command pattern | If plugin is offline, jobs sit in queue forever. | Default `expires_at = now() + 5 min`; auto-fail expired jobs; webhook on failure. | Self-rule. |
| 3 | Idempotency | Client retries POST on network blip → command runs twice. | Required `Idempotency-Key` header on POST `/commands/*`; Supabase unique index on (user_id, idempotency_key). | Stripe-style. |
| 4 | Result payload size | Some commands return MB of JSON (`get_all_clash_results_from_latest_check`). | If result > 256 KB, store in Supabase Storage, return signed URL in `result` field. | Self-rule. |
| 5 | Auth boundaries | A free-tier user calling a pro-tier command should fail before queuing. | Edge Function reads user tier from Supabase, checks command tier from registry, rejects with 402 Payment Required. | Self-rule. |
| 6 | Schema drift | `rest_api_registry.json` adds new required param → existing client requests 400. | Semver the API path: `/api/v1/...` is locked; breaking changes ship under `/api/v2/...`. Required-only fields can never be added to v1 commands; only optional. | OpenAPI versioning best practice. |
| 7 | Long-running commands | Plugin executes `resize_entire_pipe_network` for 8 min, request times out. | All command POSTs are async (return 202 + job_id); polling/SSE/webhook for result. Never synchronous. | Class Structure §REST API. |
| 8 | Plugin polling cost | Plugin polling every 1 s with 1000 users = 1000 req/s on Supabase. | Use Supabase Realtime (websockets) on `command_queue` table inserts; fall back to polling at 5 s if disconnected. | Supabase Realtime docs. |
| 9 | Replay attacks on webhooks | Bad actor replays a "command_completed" webhook to confuse client. | Webhook payload signed with HMAC-SHA256 using shared secret; include `delivery_id` and `timestamp`; reject if timestamp > 5 min old. | Stripe / GitHub webhook patterns. |
| 10 | OpenAPI spec drift | Hand-edited spec diverges from registry. | The spec is generated from registry — never hand-edited. CI gate: `npm run build && diff dist/api/openapi.json` against committed copy. | Self-rule. |
| 11 | CORS | Browser callers from unknown origins → either too permissive or too restrictive. | Public GET endpoints: `Access-Control-Allow-Origin: *`. POST endpoints: explicit allowlist (adelphos.ai + localhost in dev). | OWASP. |
| 12 | Demo abuse | Free demo endpoint hit 1M/day by a script-kiddie. | Cloudflare WAF rule + Supabase rate limit (IP) + CAPTCHA after burst threshold. | Cloudflare docs. |
| 13 | API key leakage | User commits API key to a public GitHub repo. | GitHub Secret Scanning partner program (free for issuers); auto-revoke + email user. | github.com/secret-scanning. |
| 14 | Versioning the registry itself | Adding `alwaysAvailable` to a tool changes its behaviour. | Registry includes `version: "1.0"`; consumers pin to a major; deprecations announced 90 days ahead in `/docs/changelog`. | Stripe API versioning. |

## 8. Workflow

### Workflow: Anonymous developer explores the API

1. Browse `/docs/api/` → live OpenAPI explorer.
2. `curl https://adelphos.ai/api/v1/registry | jq '.tools[] | select(.requiresBridge == "DrawingExporter")'` returns the 27 drawing exporter tools.
3. `curl https://adelphos.ai/api/v1/skills/drawing_exporter` returns the markdown skill body.
4. `curl -X POST https://adelphos.ai/api/v1/demo/run -d '{"tool":"list_rooms","sandbox":"office"}'` returns 47 room records in <500 ms.

### Workflow: Authenticated partner triggers an async command

1. Partner has API key + JWT (signed up at `/account/`).
2. `POST /api/v1/commands/export_clash_results_to_xml` with body `{ "format": "BCF 2.1" }` and headers `Authorization: Bearer <jwt>` + `Idempotency-Key: <uuid>`.
3. Edge Function validates body against `rest_api_registry.json` schema for that command. Inserts row into `command_queue`. Returns `202 Accepted` with `{ job_id, status: "queued", expires_at }`.
4. User's Revit plugin (subscribed via Supabase Realtime) wakes up. Pulls the job. Dispatches via `ExternalEvent` onto Revit main thread. Runs the command. Inserts row into `command_results`.
5. Webhook delivery worker fires registered webhooks for that user with HMAC-signed payload.
6. Partner either polled `/api/v1/commands/<job_id>/result` or received the webhook. Receives result.

### Workflow: A new `[RestApi]` command is added in MEP Bridge

See [Automation Pipeline Plan §8](../Automation%20Pipeline/Automation%20Pipeline%20Plan.md#8-workflow). One attribute, full propagation in <3 min.
