# App Page Skill

> Generates one detail page per Adelphos app (Revit Copilot, Adelphos Chat, Specbuilder, Report Builder, Document Controller, QA Manager, Schedule Builder, plus future apps).

## When to use

Whenever a new entry is added to `sandbox/data/apps.json` or an existing entry's source attributes change. Apps are first-class products on the site — every app gets its own detail page with a hero, outcomes strip, "Description" section with the brand pull-quote, hero video, the shift, features, audience pills, and an end CTA.

## Inputs (single source of truth)

`sandbox/data/apps.json` — one object per app inside the `apps[]` array.

| Field | Type | Required | Notes |
|---|---|---|---|
| `slug` | string | yes | URL slug. Lowercase, hyphenated. |
| `title` | string | yes | Display name (e.g. "Revit Copilot"). |
| `is_flagship` | bool | no | Marks the hero card on the inventory page. Exactly one app should be flagship. |
| `headline_claim` | string | yes | One bold sentence. Sits in the hero. |
| `tagline` | string | yes | Supporting line under the claim. |
| `surface` | string | yes | Where it runs (e.g. "Revit add-in · Revit 2022 → 2026 · Windows"). |
| `status` | enum | yes | `live` or `coming-soon`. |
| `icon` | string | yes | Path to icon (relative to repo root). |
| `key_outcomes[]` | array | yes (3–4) | Each `{ stat, label }`. Drives the outcomes strip. |
| `detail_paragraph` | string | yes | 80–180 words. **Quantified** time savings. Renders left of the brand quote in the "Description" section. |
| `features[]` | array | yes (8–14) | Each `{ name, desc }`. Flat features grid. |
| `feature_groups[]` | array | optional | When the features are categorised (e.g. QA Manager). Each `{ title, features[] }`. Use either `features` OR `feature_groups`, not both. |
| `the_shift` | object | yes | `{ before, after }`. Drives the side-by-side red/green callout cards. |
| `blurb` | string | yes | Short framing (rendered in the inventory tile). |
| `what_makes_it_special` | string | legacy | Kept for back-compat; no longer rendered (replaced by Features). |
| `best_for[]` | array | yes (3) | Audience pills at the bottom of the page. |
| `hero_video` | string | optional | Path to mp4. Renders under the description. |
| `primary_cta` | object | yes | `{ label, href }` — must be self-serve (Download / Run online). **No "Talk to us"**. |
| `secondary_cta` | object | yes | Same shape. |
| `seo` | object | yes | `{ why_h3, shift_h3, special_h3, who_h3 }` — intent-keyword phrases for AEO. |

## Outputs

- `dist/apps/<slug>/index.html` — generated by `scripts/build-app-pages.mjs#renderApp`
- `dist/apps/index.html` — flagship card + premium tiles, by `buildAppsInventory`

## Hard rules

1. **Self-serve CTAs only.** Apps download or run in the browser. Never "Book a walkthrough", never "Talk to us". (Agentic Services are the only place we ask for contact.)
2. **Quantify the time saving in `detail_paragraph`.** Use specific scenarios ("3 days reduced to 1 hour", "12 hours per week", "60–90 working days per year for a 5-engineer practice"). No vague "saves time".
3. **Brand pull-quote is global.** It comes from `sandbox/data/brand.json` — do **not** override per-app.
4. **`is_flagship: true` on exactly one app.** Currently Revit Copilot.
5. **Features are concrete capabilities** — what the tool actually does. Not benefits, not slogans. The user reads these to decide if it does the thing they need.
6. **Status `live` only goes live with a working CTA.** Coming-soon apps get the coming-soon pill and no live CTA target.
