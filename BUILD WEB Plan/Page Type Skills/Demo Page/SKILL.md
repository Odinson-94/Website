# Demo Page Skill

> Drives `/demos/<slug>/`. Augments the human-curated demo metadata with the
> bridge prose: 60-word hero description, 3–5 step bullets, "what you'll see
> in the chat" sample dialogue.

> **Status:** STUB. Demo metadata already exists at `data/demos.json` and the
> per-demo HTML is already generated by `build-demos.mjs`. This skill produces
> the YAML overlays at `data/demos/<slug>.yaml` that enrich the per-demo page.

## §1. Identity

You enrich a single demo with: a polished 60-word hero description, a 3–5
step bullet list of what the user does, and (when the underlying tools have
`<aiprompts>` data) a sample dialogue showing the chat the user will see.

You do not invent tools. You do not invent steps that aren't in the linked
commands.

## §2. Process

Same loading order as Tool Page Skill — see `_Skill Template/SKILL.md §3`.

Source data: ONE row from `data/demos.json` plus the linked tools from
`data/registry.json` plus the linked commands from `data/command_registry.json`.

## §3. Output

Schema is authoritative (`./schema.json`).

## §4. Hard rules

See `gotchas.md`.

## §5. When NOT to run

- Demo `status: coming-soon` → emit minimal stub, no steps invented
- Demo has no linked tools or commands → fail with `error: cannot infer steps without source tools/commands`
