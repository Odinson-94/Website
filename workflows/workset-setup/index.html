Workset Setup

Workflow 3 phases Composed from Adelphos skill

Adelphos sets up your workshared model with the standard MEP workset structure — one workset per discipline, shared worksets for grids, levels and links — then audits element assignments and moves anything that's in the wrong place.

Before you start

  • Document is open and workshared (worksharing must be enabled first)
  • User has permission to create and modify worksets
  • For move operations: elements must not be borrowed by another user

How it flows

You ask Adelphos to set up worksets — "create the standard MEP worksets" or "audit the workset assignments".

Adelphos checks what already exists: how many worksets, what they're called, who owns what, and how many elements are on each. It won't duplicate anything that's already there.

For a new model, it creates 12 standard worksets in one go: MEP_Drainage, MEP_Water, MEP_Gas, MEP_Heating, MEP_Cooling, MEP_Ventilation, MEP_Electrical, MEP_FireProtection, MEP_Controls, plus Shared_Grids, Shared_Levels, Shared_Links.

If the model already has elements, Adelphos audits them — checking that duct families are on MEP_Ventilation, pipe families on their discipline workset, and shared elements on the shared worksets. Misplaced elements are flagged with a one-click fix.

It then configures workset visibility in your view templates: each discipline template shows only its own workset plus the shared ones. The coordination template shows everything.
