Sheet Creation

Workflow 2 phases Composed from Adelphos skill

Tell Adelphos what drawings you need and it works out the sheet sizes, picks the right titleblock, creates the sheets with your numbering convention, and places views with the correct layout — single plans, plans with sections, or grid layouts for details.

Before you start

  • Views exist in the model (plans, sections, details) ready to be placed
  • At least one titleblock family loaded in the project
  • Naming convention established (or use Adelphos defaults based on existing sheets)

How it flows

You ask for sheets — "create the mechanical plan sheets" or "set up sheets for all the drainage views".

Adelphos checks what sheets already exist, what numbering convention is in use, and what views are available but not yet placed on sheets. It infers your convention from existing sheets or uses your project config.

For each view, it calculates the smallest sheet size that fits the content at the target scale — A1 for floor plans, A3 for details, or whatever your titleblock family supports. The right titleblock type is selected automatically.

Sheets are created with correct numbers following your convention (M-01-01 for mechanical Level 01, E-02-01 for electrical Level 02, etc.). Views are placed using the appropriate layout strategy: centred for single plans, plan-with-sections for coordination drawings, grid for detail sheets.

Titleblock parameters are stamped from project data: project name, client, drawn by, checked by, issue purpose. You get a summary of what was created and any views that couldn't be placed (too large for available sheet sizes, or duplicate placement detected).
