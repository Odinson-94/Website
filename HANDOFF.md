## Status

## What has been done

### Architecture
- `templates/home.html` is the source of truth for the dist home page. All CSS and HTML live there.
- `scripts/patch-dist-home.mjs` does exactly ONE thing: lifts the MEP chat block from `/index.html` (the live SPA) and injects it into `#rpChatCell` in the template, plus adds `chat.css` bundle link. Nothing else.
- `scripts/build-home.mjs` calls `patchHome()` automatically after writing `dist/index.html`.
- Build command: `node -e "import('./scripts/build-home.mjs').then(m => m.buildHome())"`
- Dev server: `node sandbox/serve.mjs` → `http://localhost:8765/dist/index.html`

### CSS bundles (consolidated)
- `css/bundles/generic.css` — resets, `:root`, `*`, `html`, `body`
- `css/bundles/shared.css` — site-wide brand, menubar, footer, dark mode
- `css/bundles/sandbox.css` — `.docs-layout`, sandbox chrome
- `css/bundles/page.css` — per-page styles (banner-card, logo-tile, etc.)
- `css/bundles/chat.css` — all chat UI (`.demo-chat-*`, `.demo-view-overlay`, etc.)
- Generated by `scripts/consolidate_css.py --write`

## User feedback patterns
- The user wants LITERAL implementations of their instructions. Do not reinterpret or "improve".
- Do not patch CSS over CSS. Edit the template directly.
- The user hates when things are rewritten instead of moved. If they say "move the chat", extract the exact block and place it — don't recreate it.
- Carousel = the "Integrates with" style from the live home page (flex-wrap cards with hover shine), NOT a CSS-only text scroller.
- The user wants to review after each 1-2 sections before continuing.
- Background was changed twice — started at #f8f9fb (too blue/cool), now at #f5f3ef (warm off-white). User may want it even warmer.
