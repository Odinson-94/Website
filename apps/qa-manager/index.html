Why use QA Manager

QA Manager runs five categories of checks continuously and surfaces them in one dashboard: drawing checks (line weights, layer naming, sheet borders, title-block compliance), schedule checks (every row reconciled against the model element it represents), specification checks (every clause tested against the live model and brief), model checks (your QA rules applied to every workset on every save), and ACC clash issues (live-imported, deduplicated and prioritised).

Every failure links one click back to the exact element, drawing or clause that needs fixing.

For a typical Stage 3 issue gate, this replaces a senior engineer's two-day pre-issue QA week with a 30-minute review — and crucially, it catches mismatches between the model and the schedule before they reach a client, which is where most expensive re-issues come from. Across a year, a five-engineer practice typically saves 40–60 working days of QA effort and avoids three to six re-issues.

Bespoke QA checks that catch what no other system would — missing maintenance space, parameter mismatches, spec-to-model drift. Runs on every save, not the week before issue.

How it works

QA is a senior engineer with a clipboard, a week before issue, finding what's broken when it's too late to fix gracefully.

QA Manager runs continuously. Drawings, schedules, spec, model, and ACC clash issues — all checked, all surfaced, all linked back to the failing element. Fix as you go, issue with confidence.

What makes it different

Other QA tools check one surface — drawings, or model, or schedules. QA Manager unifies all of them and reconciles between them: a parameter that's correct in the model but wrong in the schedule still fails, because the issued document is what counts.

Features

What QA Manager does. Each line is a shipped capability.

QA Manager features

Drawing checks · 6

Line weights Every line on every sheet checked against your house line-weight standard.
Layer naming Layer names validated against your office naming convention.
Sheet borders + title block Title block parameter completeness, project info, revision block.
Viewport scale + orientation Every viewport at the right scale; north arrows consistent.
Annotation hygiene Text styles, dimension styles, leaders all match house standards.
Sheet numbering Drawing numbers match the project numbering convention.

Schedule checks · 6

Row reconciliation Every schedule row reconciled against the model element it represents.
Missing rows Elements present in the model but missing from the schedule — flagged.
Orphaned rows Schedule rows with no matching model element — flagged.
Parameter mismatches Schedule parameter doesn't match the model parameter — flagged with both values.
Manufacturer mismatch Schedule manufacturer differs from the model element — flagged.
Cross-discipline schedules All disciplines (rad, AHU, fan, light, fixture, custom).

Specification checks · 5

Clause vs model Every clause tested against the live Revit model.
Clause vs brief Every performance clause cross-checked against the project brief.
Missing clauses Systems present in the model but uncovered by the spec — flagged.
Unsupported clauses Clauses that no longer match anything in the model — flagged.
Manufacturer alignment Spec manufacturer vs model + house preferred-list.

Model checks · 7

Workset hygiene Elements in the right workset; ownership clean.
View filter integrity View filters present, correctly applied per workset and view template.
Family parameter integrity All required parameters present and populated on every family instance.
Group instance health Group instances aren't broken / overridden.
Linked CAD cleanup Linked CAD files cleaned, not exploded, in the right view.
Naming conventions Views, sheets, families, types all match office naming rules.
Per-save runs Checks fire on every save — not just at issue gates.

ACC clash issues · 5

Live import Pulled live from your Autodesk Construction Cloud project.
Deduplication Duplicate clashes auto-merged across runs.
Severity + discipline Prioritised by severity and the disciplines involved.
Element-link One click jumps to the offending element in Revit.
Resolution status Tracks which clashes are resolved, ignored, or pending.

Cross-surface reconciliation · 4

Model vs schedule vs spec When the three disagree, the issued documents win — fail flagged.
Model vs calc Element output (heat, flow, lux, dB) matches the calc spreadsheet.
Pre-issue gate Single comprehensive sweep before any drawing leaves the office.
Continuous dashboard Live status page for every project, not a one-shot report.
