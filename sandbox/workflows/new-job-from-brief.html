New job from brief to Stage 2

From a fresh client brief to a coordinated Stage 2 model in one afternoon. This workflow chains seven demos and twelve tools end to end.

Workflow 5 phases Live

Phases

1. Set up the project from template

Open BUILD MEP, pick the JPA master template for your discipline mix, point it at the project folder.

video placeholder · workflows/new-job-from-brief/phase-1.mp4
  1. Pick the master template
  2. Confirm project info (number, address, RIBA stage)
  3. Templates, filters, sheet sizes auto-applied
Tools: setup_project_from_template · list_filters · apply_view_template_to_views_by_filter

2. Pull in the briefing data

Drop the client brief PDF, the architect's plan PDF, and any site survey notes into the project intake folder.

video placeholder · workflows/new-job-from-brief/phase-2.mp4
  1. Drop PDFs into /01 Intake/
  2. BUILD MEP extracts room schedules, fixture counts, design intent
  3. Open Briefing → review and confirm
Tools: parse_brief_pdf · extract_room_schedule_from_pdf · seed_briefing_module

3. Generate the MEP strategy report

One prompt, one document. The strategy report is drafted from the brief, ready to review and issue.

video placeholder · workflows/new-job-from-brief/phase-3.mp4
  1. Ask: "Generate the MEP Strategy report for this project"
  2. Review draft, edit inline
  3. Export to .docx and PDF
Tools: generate_mep_strategy_report · export_report_to_word

4. Set up the model — views, sheets, schedules

Run the model-setup demos in sequence: views, then templates. Sheets and schedules are wired from the discipline mix.

video placeholder · workflows/new-job-from-brief/phase-4.mp4
  1. "Set up views for all disciplines on all levels"
  2. "Apply standard view templates"
  3. "Generate the Stage 2 sheet set"
Tools: setup_views_per_discipline_per_level · apply_view_template_to_views_by_filter · generate_sheet_set_for_stage

5. Run end-of-stage QA pack

Before issue, run the full QA: drawings, specification, schedules, model warnings, clashes. Output is one PDF + one BCF.

video placeholder · workflows/new-job-from-brief/phase-5.mp4
  1. "Run the Stage 2 QA pack"
  2. Review issues, mark false positives
  3. Export QA report and BCF
Tools: run_qa_pack_for_stage · export_clash_results_to_xml (REST) · export_qa_report_to_pdf
