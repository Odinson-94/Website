Apply View Template

writes to model Public API · not yet exposed

Applies a named view template to a target view using SetTemplate.ApplyTemplate. If no viewId is provided, the active view is used as the target. The template is resolved by exact name match against all view templates in the project. Related: AddFilterToTemplateCommand for adding filters to templates before applying them.

Before you run this

Active document must be open. The named view template must exist in the project. The target view must not itself be a template (cannot apply a template to a template).

After it runs

The target view's ViewTemplateId is set to the resolved template's element ID. All template-controlled settings are applied to the view.

When to use this

  • User wants to apply a standard template to the current view for consistent display.
  • AI is configuring views as part of automated sheet setup or coordination workflows.
Don't use this when

Creating new view templates (use a dedicated CreateViewTemplateCommand). Adding filters to templates (use AddFilterToTemplateCommand). Removing templates from views.
