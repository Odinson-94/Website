Launch Document Controller

writes to model Public API · not yet exposed

Launches the standalone Document Controller WPF desktop app from a Revit ribbon button. The Document Controller is NOT hosted inside Revit — it runs as its own process (DocumentController.WPF.exe) and shares the Supabase backend. The command resolves the exe path from an environment variable override, then the published add-in folder, then known development build-output locations. If an instance is already running, it is brought to the foreground instead of launching a duplicate. Uses ErrorPanel for the three-wire error model and CommandFeedback for user notifications.

Before you run this

DocumentController.WPF.exe must have been built at least once (dotnet build DocumentController/DocumentController.WPF/DocumentController.WPF.csproj) OR be deployed alongside the MEPBridge add-in in the ApplicationPlugins folder.

After it runs

A DocumentController.WPF window is visible on the desktop. If a prior instance existed, that window is brought to the foreground. If the exe cannot be found, a recoverable error is raised listing every probed path for the user.

When to use this

  • User clicks the "Doc Controller" ribbon button to open the standalone document management desktop app while continuing to work in Revit.
Don't use this when

Hosting the Document Controller inside Revit (it is intentionally a separate process so it can run without Revit and share the web/desktop chat core). Launching from headless automation — requires a Windows desktop session.
