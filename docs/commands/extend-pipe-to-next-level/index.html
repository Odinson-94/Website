Extend Pipe To Next Level

Drainage writes to model Public API · not yet exposed

Command: Extends a selected pipe upwards to the next primary level. INCLUDES LEVELS FROM LINKED MODELS (architectural links, etc.) WORKFLOW: 1. User selects a pipe endpoint 2. Command uses ListLevels.GetPrimaryLevels() to find level structure (host + linked models) 3. Determines current pipe position and next level above 4. Creates extension pipe to next primary level + 150mm offset 5. Matches all properties from source pipe 6. Optionally creates bend fitting at connection

Before you run this

Active document must be open. At least one pipe must exist in the model. At least two primary levels must exist (current and target). User must select a valid pipe element.

After it runs

A new pipe is created extending from the selected pipe to the next primary level. Extension terminates at level elevation + 150mm offset. New pipe matches system, size and family routing types. Bend fitting created at connection point (optional).

When to use this

  • User needs to extend an SVP or drainage riser to the next floor.
  • User wants to grow vertical stacks through floor plates.
  • User is extending vent pipes through ceiling/floor assemblies.
Don't use this when

Not for autonomous AI execution. This command requires interactive Revit UI input (dialogs, picks, or manual confirmation).

Side effects

Creates a new pipe element in the model. May create a bend fitting element.
