Select By Filter

writes to model Public API · not yet exposed

Selects elements matching a named ParameterFilterElement. Resolves the filter by name, extracts categories via GetCategories() and rules via GetElementFilter(), then applies to all elements in the chosen scope. Uses SelectByFamilyNameCommand.BuildFamilyTypeBreakdown for result summary. Called by CommandExecutor when Claude invokes execute_command. Related: SelectByCategoryCommand for category-only selection, SelectByParameterCommand for deep parameter search.

Before you run this

Document must contain at least one ParameterFilterElement. Active document and view must be open.

After it runs

Selection is replaced with elements that pass the filter rule set. Returns JSON with filterName, scope, totalSearched, totalSelected, and breakdown.

When to use this

  • Select all elements currently targeted by "Coordination - Clash" filter.
  • Select elements matching a discipline-specific filter before export.
  • Verify which elements a view filter affects before toggling visibility.
Don't use this when

Creating or editing filters (manual Revit operation). Selecting by category alone (use SelectByCategoryCommand — faster). Deep parameter search across all fields (use SelectByParameterCommand).
